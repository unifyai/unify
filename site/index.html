Steerable AI teammates.

unity-agent is Unify's persistent AI teammate, open-sourced. It runs on your machine, plans in real Python over typed state, steers live work without restarting, and coordinates with other teammates over shared typed state.

Features

01

One identity across channels

SMS, email, phone calls, meetings, Slack, WhatsApp. One identity, one memory, one task queue across every channel.

voice · sms · email · slack · meet
02

Consolidates as it runs

Every 50 messages, the MemoryManager runs a background extraction pass — distilling contacts, tasks, preferences, and policies into typed tables. Reusable Python functions and guidance procedures are stored alongside.

memory · functions · guidance
03

Durable scheduled tasks

Scheduled work that returns the same live handle as everything else. Pause, defer, or interject a task mid-run — from anywhere, while it’s still executing.

scheduler.defer(task, until="fri")
04

Steerable nested execution

Every operation returns a live handle. Ask for status, interject, pause a branch, or stop one task without affecting the others.

handle.interject("also check trains")
05

Code plans over typed primitives

CodeActActor writes Python over typed primitives.* in a sandboxed session. One program per turn, with variables, loops, and composition.

for c in primitives.contacts.ask(...)
06

Hive mind, opt-in

Typed state managers can share state across assistants — contacts, tasks, transcripts, knowledge. Opt-in per manager, scoped by workspace, auditable at the boundary.

share: contacts + tasks · scoped

Install

Runs locally against simulated backends. No platform setup required for the sandbox.

~/unity 
# grab the three repos
git clone https://github.com/unifyai/unity.git
git clone https://github.com/unifyai/unify.git
git clone https://github.com/unifyai/unillm.git

# sync the env (uv recommended)
cd unity && uv sync

# wire up your keys
cp .env.example .env

# launch the sandbox
source .venv/bin/activate
python -m sandboxes.conversation_manager.sandbox \
    --project_name Sandbox --overwrite

At the prompt, select option 2 — CodeAct + Simulated Managers. This runs the full architecture against simulated backends.

Architecture

01 orchestration

ConversationManager

Owns the session. Sees notifications, in-flight work, and user input. Makes a single deliberate decision per turn. Talks to the fast brain over IPC when voice is live.

02 planning

CodeActActor

Writes Python against typed primitives in a sandboxed session. One program per turn, with variables, loops, and composition. Discovery-first tool policy ensures the actor explores stored functions and guidance before writing new code.

03 domain

State Managers

Each manager owns its domain and runs its own async tool loop. ContactManager. KnowledgeManager. TaskScheduler. Each returns a live handle. Each can delegate further.

04 collective

Shared typed state

Managers can selectively share state across assistants. Cross-assistant coordination is opt-in per manager, scoped by workspace, and auditable at the boundary.
