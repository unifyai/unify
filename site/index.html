What it is

Not another chatbot wrapper.
A runtime with opinions.

01

One mind across every channel

SMS, email, phone calls, meetings, Slack, WhatsApp — one identity, one memory, one task queue across every surface. What gets said on Monday's call is sitting in Tuesday's email thread.

voice · sms · email · slack · meet
02

Consolidates as it runs

Every 50 messages, the MemoryManager runs a background extraction pass — distilling contacts, tasks, preferences, and policies into typed tables. Plus reusable Python functions and stored guidance. Compounding structure, not compounding context.

memory · functions · guidance
03

Runs in the background

Durable scheduled tasks you can pause, defer, or interject mid-run. Not just cron — live handles you can steer from anywhere, even while they’re executing.

scheduler.defer(task, until="fri")
04

Steerable all the way down

Every operation returns a live handle. Ask it for status. Interject mid-run. Pause a branch. Stop one task without burning the other three. No restarts, no lost context.

handle.interject("also check trains")
05

Code plans, not tool menus

CodeActActor writes real Python over typed primitives.* in a sandboxed session. You get variables, loops, composition, reuse — not a thousand JSON tool-calls in a trenchcoat.

for c in primitives.contacts.ask(...)
06

Many minds, one world

Typed state managers can share state across assistants — contacts, tasks, transcripts, knowledge. Optional, scoped, auditable. Hive-mind energy, without the tyranny.

share: contacts + tasks · scoped

Try it locally

Clone. Sync. Run the brain.

Runs on your machine. No platform install required to poke at the runtime. Five minutes to a steerable, nested, code-first sandbox.

~/unity 
# grab the three repos
git clone https://github.com/unifyai/unity.git
git clone https://github.com/unifyai/unify.git
git clone https://github.com/unifyai/unillm.git

# sync the env (uv recommended)
cd unity && uv sync

# wire up your keys
cp .env.example .env

# launch the sandbox
source .venv/bin/activate
python -m sandboxes.conversation_manager.sandbox \
    --project_name Sandbox --overwrite

At the prompt, pick option 2 — CodeAct + Simulated Managers. That's the full runtime shape with simulated backends, which is what you want the first time.

How it runs

The stack, top to bottom.

01 orchestration

ConversationManager

Owns the session. Sees notifications, in-flight work, and user input. Makes a single deliberate decision per turn. Talks to the fast brain over IPC when voice is live.

02 planning

CodeActActor

Writes Python against typed primitives. Runs in a sandboxed session. One program per turn, with variables and control flow. Discovery-first tool policy keeps it honest.

03 domain

State Managers

Each manager owns its domain and runs its own async tool loop. ContactManager. KnowledgeManager. TaskScheduler. Each returns a live handle. Each can delegate further.

04 collective

Shared substrate

Managers can selectively share state across assistants. Typed boundaries make cross-assistant coordination legible instead of terrifying. Opt-in. Scoped. Auditable.
