# Slim runtime requirements for `python -m unity.gateway` (the
# Phase B/C gateway aggregator) in container deploys.
#
# Excludes the heavy assistant-runtime deps (pyaudio, browser-use,
# playwright, spacy, docling, livekit-agents, opencv, pymupdf, etc.)
# so the comms-app image stays buildable on python:3.12-slim without
# system-level compilers or audio libs.
#
# Use:
#   pip install --no-deps /path/to/unity   # unity package itself
#   pip install -r /path/to/unity/requirements-gateway.txt
#
# Why a requirements file and not a [gateway] pyproject extra:
# pip's resolver re-evaluates base deps when installing extras, so
# `pip install unity[gateway]` still tries to satisfy pyaudio etc.
# A requirements file lets the Dockerfile install the gateway dep
# set explicitly without going through unity's full base-deps
# resolution.
